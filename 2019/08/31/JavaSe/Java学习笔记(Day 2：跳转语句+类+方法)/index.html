类（非常重要）

1. 类与对象

  1. 类定义了一种新的数据类型（引用数据类型），一旦定义了一个类，就可以使用这种新的类型创建该类型的对象。类是对象的模板，对象是类的实例。
  2. 在类中定义的数据或变量称为实例变量（instance-var）,实现算法的代码包含在方法中。在类中定义的方法和变量都被成为类的成员。大多数类中，实例变量由为该类定义的方法进行操作和访问(使用构造函数和方法，尤其是private实例变量，需要构造函数来实例化，要用方法来return返回值)。方法决定了类中数据的使用方式。
  3. （个人理解：比如说人是一个类，我就是人这个类的一个实例（或者说是人这个类的一个对象），那么我的身高、体重这些变量都是唯一的数据，只属于我的数据（或者说变量）。
  1. 一个简单的类：

上面的例子定义了一种新的数据类型为Box类引用数据类型。上面的例子只是创建了一个模板，还没有创建实际的对象（也就是没有创建具体是哪一个盒子是Box类的对象）。创建一个Box对象的语句如下：Box mybox = new Box();
执行上面这条语句后，mybox就变成了Box类的一个实例（mybox包含了Box类的width、height、depth等实例变量）。为了访问这些变量，需要使用(.)运算符。点运算符将对象的名称（mybox）和实例对象的名称（width、height、depth）链接在一起。例如：mybox.width = 100;
表示将mybox对象中包含的width变量的副本(副本指的是从Box类中复制过来的实例变量)赋值为100。

Box类的完整程序：

  1. 每个对象都有自己独一无二的实例对象副本，一个类里的两个对象的实例对象副本互不影响:

2. 声明对象

一般形式：在声明的同时实例化对象：Box mybox = new Box()，右边的new Box(),是以Box为模板，在堆空间里创建一个Box类对象，Box()指调用构造方法；左边的Box mybox指的是创建了一个类的引用对象，“=”把引用对象指向刚创建的Box()对象，mybox才正式成为了Box的对象
或分成两步：先声明：Box mybox; 再实例化：mybox = new Box();
第一行代码声明mybox为Box类型对象的引用，第二行代码将Box类型的实例变量赋值给mybox，mybox成为了Box的对象，new运算符给对象分配了内存

  1. 方法返回值的类型(也就是width、height和depth的类型）必须和方法指定的返回类型兼容(double volume())
  2. 用于接收方法返回值的变量(vol的类型),也必须和方法指定的返回类型(double volume())兼容

3. 带参数的方法

当执行mybox1.dimension(50,50,50)时，void dimension方法在(parameter-list)部分会把50分配给参数w、把50分配给参数h、把50分配给参数d，然后在方法内部通过width = w、height = h、depth = d把值赋给实例变量。

构造函数（构造函数是一种特殊的方法）

1. 使用构造器时需要记住：

  1. 构造器必须与类同名（如果一个源文件中有多个类，那么构造器必须与公共类同名）
  2. 每个类可以有一个以上的构造器
  3. 构造器可以有0个、1个或1个以上的参数
  4. 构造器没有返回值
  5. 构造器总是伴随着new操作一起调用。构造函数的作用是在创建对象之后立即初始化对象,构造函数的名称和包含它的类名称相同，这就是为什么创建（实例化）对象的语句是：类名 对象 = new 类名()；因为在执行这个语句的时候，实际发生的操作是调用类的构造函数。java会为类默认创建构造函数。这就是为什么明明在之前的程序中都是用：类名 对象 = new 类名()这一语句来实例化对象，又没有构造类函数，但是却能正常执行的原因。（在没有创建构造函数的程序中，java为我们创建了一个无参构造函数(方法)，无参构造函数是所有构造函数的父类。
    再次以Box示例：（参数化的构造函数）

当运行 Box mybox1 = new Box(50,40,30)语句时，调用了Box()方法，先把50、40、30传递给形参w、h、d，而后Box()方法内部再把w、h、d的值赋给width、height、depth。
