this关键字：主要用于区分成员变量和局部变量，指代本类的同名对象

成员变量

1. 类变量和实例变量:成员变量是在类范围内定义的变量。而成员变量又可细分为类变量和实例变量（也有人称变量为属性，其实是同一个东西）。实例变量就不用static修饰的，如之前Box类里的width、height和depth：

  1. 实例变量：

width、height、depth就是成员变量里的实例变量，虽然我们只是声明了，但是java会默认为成员属性进行初始化，给定一个初始值0.实例变量是随着对象的存在而存在的，也就是说只有当对象调用它的时候它才有意义(也就是有实参传入)。

2. 局部变量:局部变量是在一个方法内定义的变量，只能作用与定义它的这一作用域。局部变量可分为形参、方法局部变量、代码块局部变量。

  1. 形参：

上面dimension方法中的parameter-list中定义的w、h和d就是一个形参，形参可以不用初始化，它用于接收对象传入的实参。w、h、d这三个形参只在dimension这个方法内有效。

  1. 方法局部变量：

同样的，dimension代码块内的width、height、depth就是方法局部变量,只在dimension这个方法内有效

  1. 代码块局部变量：

比如上面代码中的vol就是一个代码块局部变量，只在BoxDemo这个类中有用

static关键字

  1. 静态变量（类变量）具备以下特点：
    1. 随着类的加载而加载:在创建类的任何对象之前访问该成员，并且不需要任何对象引用。
    2. 优先于对象存在
    3. 被所有对象所共享
    4. 可以直接被类名调用
    5. 静态代码块只执行一次，且优先于静态方法（static void main())被执行。

final关键字

  1. 可以将变量声明为final，final变量必须在声明时进行初始化。有两种方式实现：第一种，在声明时为其赋值，如：final int a = 10。第二种，在构造函数中为其赋值(形如static变量，只不过static变量使用方法返回得到值，而final变量则是在构造函数中通过形参来为其赋值)
  2. 方法参数和局部变量也可以声明为final。可以防止其被多次赋值

super关键字:当子类需要引用他的直接超类时，可以使用关键字super.

1. 使用super调用超类的构造函数

本例中width、height、depth被设置为私有，无法通过子类直接引用。所以只能通过调用定义其的超类的构造函数将其初始化
每次Boxadd实例化对象时，都会调用对应的Boxadd构造函数，而Boxadd构造函数中包含了Boxadd这个子类的直接超类(也就是Box类)的super方法，所以又会根据super()方法传入的参数匹配调用Box的构造方法。如执行：Boxadd mybox1 = new Boxadd(10,20,30,5)时，调用Boxadd(double w,double h,double d,double m){}这个构造函数，而构造函数内部又有super(w,h,d)方法，传入了10，20，30这三个参数，将会匹配调用Box(double w,double h,double d){}构造函数，将width、height、depth这三个私有变量初始化为10，20，30。
