今日总结：

  1. 泛型创建：class Gen,T为类型参数，是占位符，当创建对象时，将实际类型传递给Gen

  2. 泛型实例化：
    第一种方式：Gen<类型> ob = new Gen<类型>(parameter-list)；
    第二种方式：Gen ob = new Gen(parameter-list);
    第三种方式(当泛型中构造函数的参数为多个类型参数):Gen<?> ob = new Gen<>(parameter-list1,parameter-list2…)

  3. 泛型中可以重载构造函数(有多个不同版本的构造函数)：实例：

  4. 泛型可以有两个类型参数：Gen<T,V>{},创建对象时必须传入两个类型：Gen<Integer,String> ob = new Gen<Integer,String>(10,”String传递给V”)。

  5. 指定T的类型（设置上界）：Gen

  6. 通配符参数：boolean meth(Gen<?> ob){},“？”表示未知类型，表示和任意类型的ob进行匹配

  7. 泛型方法：声明形式：(修饰符)+类型参数(<T （extends）>)+返回类型，实例：static <T extends Comparable,V extends T> boolean isTn(T x,V[] y)，调用：isTn(2,nums),直接传递类型值

  8. 泛型构造函数(可以将构造函数泛型化，即使它们的类不是泛型类)：声明形式：类型参数+构造函数名+(参数类型)，实例：Test(T arg)

  9. 泛型接口：声明形式：interface+接口名+类型参数。实现接口类的声明：class B<T extends Comparable> implements A{}

  10. 泛型超类：在泛型层次中，类层次的所有子类都必须向上传递超类所需要的所有类型参数(与子类的构造函数通过super方法调用构造函数类似):
    class Gen2<T,V> extends Gen{
    V ob1;
    Gen(T o){

    super(o);

    }
    }

  11. 非泛型类作为泛型超类：
    class Gen extens A{
    T ob;
    Gen(int i,T o){

    super(i);
ob = o;

    }
    }

  12. 重写泛型方法（两种情况）：
    一、子类泛型类初始化父类：（在实例化对象时指定T的类型）

二、子类声明时制定了父类的类型(子类不是泛型类)：class B extends A

泛型:操作参数化类型的类、接口或方法称为泛型

1. 创建泛型类：

  1. 创建class Gen:T是类型参数的名称，创建对象时将实际类型传递给Gen，只要声明类型参数，就需要在尖括号中指定。应用：Gen iob；该语句相当于：Integer iob。
  2. 声明T ob：如果将String类型传递给T，ob将是String类型，ob和T的类型一定一致
  3. 构造函数Gen (T o){ob=o;}：参数o的类型为T,这意味着o的实际类型取决于创建Gen对象时传递给T的类型。因为ob的类型也是T，所以ob和o类型一致
  4. T指定方法的返回类型T getob(){return ob;}:因为ob也是T类型（前面声明了）,所以这样也是合法的。
  5. showType方法：ob对象调用getClass()方法获得Class对象(java.(…)，Class对象再调用getName()方法来显示T的类型
  6. 泛型必须独立放在一个文件中

2. 使用泛型类

  1. Gen iob:创建了完整版的Gen类,将Integer类型传递给T。ob变成了Integer类型
  2. iob = new(88):调用了构造函数，传入Integer参数88，ob = 88
  3. iob.showType()、iob.getob():对象直接调用方法

3. 带两个类型参数的泛型类

4. 创建泛型方法

  1. 可以声明本身使用一个或多个参数类型的泛型方法
  2. 可以在非泛型类中创建泛型方法
  3. 形式：(修饰符)+类型参数+返回值类型+方法名

5. 泛型构造函数

  1. 可以将构造函数泛型化，即使它们的类不是泛型类
  2. 形式：类型参数+构造函数名+(参数类型)

指定了一个泛型类型的参数，并且这个参数必须是Number的子类，所以可以使用各种数值类型来调用构造函数。

6. 泛型接口

分析：

  1. 接口A的声明方式：interface A<T extends Comparable>{}.
    类型参数为T，上界是Comparable，指定了进行比较的对象类型：vals[i].compareTo(v),vals[i]的基类型为T，v的类型也为T
  2. 实现接口类的声明：class B<T extends Comparable> implements A.类名+<上界限制>+implements+泛型接口名(A).
    因为接口A需要实现Comparable泛型接口的方法，所以实现接口A的类必须指定相同的界限。

7. 有界类型

8. 有界通配符:有界通配符为类型参数指定上界或下界，从而限制方法能够操作的对象类型，通过extends指定上界。建立上界：<? extends superclass>。建立下界：<? super subclass>

9. 泛型类层次（继承泛型类时，必须对父类中的类型参数进行初始化。）

  1. 用子类中的泛型类型初始化父类：
    public class Son extends Parent{}
  2. 用具体类型初始化：
    public class Son extends Parent{}
    在泛型层次中，类层次的所有子类都必须向上传递超类所需要的所有类型参数(与子类的构造函数通过super方法调用构造函数类似)

10. 使用泛型超类

10. 泛型子类：非泛型类可以作为泛型的超类

11. 重写泛型类的方法

  1. 子类通过具体类型初始化泛型类

  2. 子类的泛型类初始化父类

12. 泛型的一些限制：

  1. 不能实例化类型参数：ob = new T() //T只是一个占位符，编译器不知道T具体为什么类型
  2. 静态成员：静态成员不能在使用在类中声明的类型参数：static T ob;//非法； static T getob(){} //非法
  3. 泛型数组：不能实例化元素类型为类型参数的数组：T arry[] = new T [3]; //T的具体类型未知
  4. 泛型类不能拓展Throwable
