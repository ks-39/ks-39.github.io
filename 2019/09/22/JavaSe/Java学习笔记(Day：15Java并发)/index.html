P920-P970

内容：

  1. 并发API包：
    1. java.util.concurrent:
      1. 同步器:提供了同步多线程交互的高级方法
        1. Semaphore类：（设置许可证来控制并发）（只有获得许可的线程可以执行）
          1. Semaphore sem = new Semaphore(permit-nums);设置可获取的许可数量
          2. sem.acquire():获取许可，在释放许可之前其他线程无法执行
          3. sem.release():释放许可，其他线程可以争抢许可
        2. CountDownLatch类：（通过设置需要触发的事件数来控制）
          1. CountDownLatch cdl = new CountDownLatch(event-nums);设置需要触发的事件数量
          2. void.countdown();每调用一次countdown方法，事件数-1
          3. void await()；可以指定某个线程来需要触发事件
        3. CyclicBarrier类：（设置需要执行线程数）
          1. CycliBarrier cb = new CycliBarrier(thread-nums,Runnable r);执行nums个线程后实例化r线程
          2. void await()；可以指定某个线程需要等待触发结束
        4. Exchanger类：（线程间通信）
          1. Exchanger ex = new Exchanger();声明类型参数
          2. ex1.exchange(Obj obj);线程1传送数据
          3. ex2.exchange(new Class())；线程2接收数据并先建类型来存储
        5. Phaser类：（阶段进行）
          1. Class Myphaser extends Phaser(){};继承Phaser类
          2. protected boolean onAdvance(){}; //重写onAdvance()方法；（return false表示进行下一阶段，return true表示结束）
          3. 在线程类的构造函数中调用register()方法：ph.register();
          4. 在run()方法中调用ph.arriveAndAwaitAdvance()方法执行阶段并等待阶段结束
      2. 执行器：管理线程执行
        1. Executor接口：
        2. ExecutorService接口：
          1. 创建同步器：
          2. 创建线程池：ExecutorService es = Executors.newFixedThreadPool(int num);
          3. 启动线程：es.execute(Runnable t);相当于调用了start()方法，能够启动run()
          4. 如果同步器需要进行等待就执行await()
          5. 关闭线程池:es.shutdown();
        3. Callable()接口：被实现
        4. Future()接口：用于声明类型参数
          1. 实现Callable接口:class A extends Callable{} //指定类型参数
          2. 重写call()方法：public void call(){}
          3. 创建线程池：ExecutorService es = Executors.newFixedThreadPool();
          4. 声明Future接口的类型参数:Future f1;
          5. Future接口引用接收es调用submit实例化Callable实现类：f1 = es.submit(new Sum(10));
          6. Future接口调用get()方法读取call()方法返回值：
      3. 并发集合：
      4. Fork/Join框架：（重点）
        1. ForkJoinTask
        2. ForkJoinPool：执行和管理线程
        3. RecursiveTask：返回结果
          1.
        4. RecursiveAction：不返回结果

1. 同步器：

  1. Semaphore类：(实现了经典的信号量)(通过计数器控制访问)
    1. 构造函数：
      1. Semaphore sp = new Semaphore(int num); //指定初始许可证计数大小
      2. Semaphore sp = new Semaphore(int num,boolean how)； //将how设置为true，确保线程以顺序访问
    2. 获取许可证：acquire()
      1. sp.acquire() throws InterruptedException //获取一个许可
      2. sp.acquire(int num) throws InterruptedException //获取num个许可
    3. 释放访问：release()
      1. sp.release();

生产与消费：

  1. CountDownLatch类：（通过设定等待事件和调用await()来顺序启动线程）

    1. 构造函数：
      1. CountDownLatch cdl = new CountDownLatch(int num)； //需要等待num个事件
    2. 等待解锁器：
      1. void await() throws InterruptedException
      2. boolean await(long wait,TimeUnit tu) throws InterruptedException
    3. 触发事件:
      1. void countDown(); 每调用1次，num-1

  2. CyclicBarrier类：（线程组在指定执行点等待）

    1. 构造函数：
      1. CyclicBarrier(int numThread); //指定线程数量
      2. CyclicBarrier(int numThread,Runnable action); //在num个线程执行完后执行线程action
    2. 等待线程：
      1. int wait() throws Exception

  3. Exchanger类：（简化线程间交互）（泛型类）

    1. 声明：
      1. Exchanger //V指定了需要交换的数据类型
    2. exchanger()方法：
      1. V exchanger(V obj) throws InterruptedException

  4. Phaser类：（允许一个或多个线程进行同步）

    1. 构造函数：
      1. Phaser ph = new Phaser()
      2. Phaser ph = new Phaser(int numparties)
    2. 常用方法：
      1. int register()；注册party
      2. int arriveAndAwaitAdvance();等待，直到所有party到达
      3. int arriveAndDeregister();线程到达时注销本身
      4. final int getPhaser(); 获取阶段编号
    3. 步骤：
      1. 创建Phaser实例
      2. 注册party

2. 执行器：（启动并控制线程）

  1. Executor接口：

    1. 启动指定线程方法：
      1. void execute(Runnable thread);//如此一来，不需要start()方法也能启动run()方法
    2. ExecutorService接口继承了Executor接口
      1. 停止调用：es.shutdown();
    3. 创建线程池：
      1. Executors.newFixedThreadPool(int num);//设置可共享线程数量

  2. 步骤：

    1. 创建CountDownLatch实例：CountDownLatch cdl = new CountDownLatch(int eventnums)
    2. 创建线程池：ExecutorService es = Executors.newFixedThreadPool(Plevel);
    3. 启动线程：es.execute(Runnable t);
    4. 等待事件：cdl.await()
    5. 关闭线程池：es.shutdown()

  3. Callable接口和Future接口

    1. Callable接口：（泛型接口）
      1. interface Callable //需要被类实现
      2. call()方法：V call() //需要被重写,返回值类型与指定的类型参数一致，返回的值能够被对应的Future接口引用通过get()方法读取
    2. Future接口：（泛型接口）
      1. interface Future //声明类型参数
      2. get()方法：V get() //读取调用的Callable实现类中call()的返回值
    3. 步骤：
      1. 实例化ExecutorService对象,创建线程池：ExecutorService es = Executors.newFixedThreadPool(Thread num);
      2. 声明Future接口引用，指定类型参数:Future f1;
      3. Future接口引用接收es调用submit实例化Callable实现类：f1 = es.submit(new Sum(10));
      4. Future接口调用get()方法读取call()方法返回值：
        f1.get();
      5. 关闭线程池：es.shutdown();

3. Lock锁：java.util.comcurrent.locks

4. Fork/Join框架（并行）：java.util.concurrent

  1. ForkJoinTask类:定义任务
  2. ForkJoinPool：管理ForkJoinTask的执行
  3. RecursiveAction：ForkJoinTask的子类,用于不返回值的任务
  4. RecursiveTask:ForkJoinTask的子类，用于返回值的任务

    ForkJoinTask:(核心方法fork()和join())

  5. ForkJoinTask与Thread不同，ForkJoinTask表示任务的轻量级抽象，而不是执行线程。
  6. ForkJoinTask通过线程池来执行
  7. 核心方法：
    1. final ForkJoinTask fork(); //为调用任务的异步执行提交调用任务，意味着调用fork()方法的线程将持续运行
    2. final V join(); //等待调用该方法的任务终止，结果返回
    3. final V invoke();将fork和join合并。
    4. static void invokeAll(ForkJoinTask<?> task1,….);执行所有指定的任务

      RecursiveAction类：（ForkJoinTask的子类）(封装不返回结果的任务)

  • 使用步骤：
  1. 创建实现该类的类：class ReA extends RecursiveAction{}
  2. 定义compute()方法：protected void compute(){}
  3. 创建ForkJoinPool线程池：ForkJoinPool fjp = new ForkJoinPool(int thread-nums)
  4. 创建实现类对象：A task = new A()
  5. ForkJoinPool对象调用invoke()方法：fjp.invoke(task)

    实例:使用5个线程并行输出0-100：

RecursiveTask类：（ForkJoinTask的子类）（封装返回结果的任务）

  1. 创建实现该类的类：class ReT extends RecursiveTask{}
  2. 定义compute()方法：protected Class-type compute(){}
  3. 创建ForkJoinPool线程池：ForkJoinPool fjp = new ForkJoinPool(int thread-nums);
  4. 声明ForkJoinTask类型参数：ForkJoinTask task;
  5. 将ForkJoinTask对象指向实现类对象：task = new A()
  6. ForkJoinPool对象fjp调用submit()方法：fjp.submit()
  7. task调用get()得到返回值：task.get();

    实例：简单计算0-10000的和

ForkJoinPool类：（管理ForkJoinTask任务）

  1. 构造函数：
    1. ForkJoinPool fjp = new ForkJoinPool();
    2. ForkJoinPool fjp = new ForkJoinPool(int plevel); //设置支持并行的线程数
  2. 常用方法：
    1. fjp.invoke(ForkJoinTask task); //开始task任务，并返回任务结果。等待结果返回
    2. fjp.execute(ForkJoinTask task); //开始task任务，不需要等待返回
  3. 公共池：（如果没有构建ForkJoinPool对象，默认使用公共池)

    分而治之原理

  4. 原理：对于包含N个整数的数组，可以将其中的元素分给两个任务处理。每个任务处理N/2个元素。而每个子任务又可以将其分给另一组任务。直到达到某个临界点。
  5. 正确设置临界点：任务应当在100到10000个计算步骤之间的某个位置执行
