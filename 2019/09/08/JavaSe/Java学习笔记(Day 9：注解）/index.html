注解（import java.lang.Annotation)

  1. 注解是通过基于接口的机制创建的，所有类型的声明都可以有关联的注解。如：类、方法、域变量、参数以及枚举常量。
  2. 注解要放在声明的最前面
  3. 注解无法继承，因为它拓展了Annotation接口，重写了hashCode()、equals()和toString()方法
  4. 应用注解时，通过使用成员名称来为注解的成员提供值来初始化。除了：有default或单成员value或者多成员除value外都有default的情况

指定保留策略:(@Retention(Retention.type))

  1. SOURCE:编译期间被抛弃（也就是不能获取）
  2. CLASS:编译时存储到.class文件（也是无法获取）
  3. RUNTIME:编译时存储到.class文件，可以被获取

利用反射获取注解

  1. 获取Class对象：Test ob = new Test(); Class c = ob.getClass(); Class为泛型特性，c是引用变量，ob.getClass()表明希望获取ob对象的类的注解信息
  2. 然后就可以获取某个方法的注解（c.getMethod())，某个作用域的注解(c.getField()),某个构造函数的注解(c.getConstructor())

为了获取带有参数的方法，必须指定表示参数类型的类对象（String.class和int.class,String和int为参数类型，class为类对象)作为getMethod()方法的参数

内置注解

  1. @Retention：用于指定保留策略（只能注解其它注解）
  2. @Documented:标志接口，注解被文档化（只能注解其它注解）
  3. @Target：用于指定应用注解（也就是把注解当成一种标签代替某些关键字），只能用于注解其它注解。参数只能是ElementType枚举的常量，指定了应用类型：ANNOTATION_TYPE(另一个注解)、LOCAL_VARIALBE(局部变量)、PACKAGE(包)、TYPE(类、接口或枚举)、TYPE_PARAMETER(类型参数)、TYPE_USE(类型使用)。可以指定一个或多个：@Targer({ElementType.TYPE_USE,ElementType.TYPE_PARAMETER})
  4. @Inherited：标记注解，只能用于另一个注解声明。
  5. @Override：标记注解，只能用于方法，带有@Override注解的方法必须重写超类中的方法
  6. @FunctionalInterface：被注解的接口为一个函数式接口

类型注解

类型注解必须包含ElementsType.TYPE_USE作为目标，放在需要应用该注解的类型的前面
