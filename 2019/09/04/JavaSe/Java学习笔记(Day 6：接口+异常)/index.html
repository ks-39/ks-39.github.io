P200-P237

今日内容：

  1. 接口：
    1. 定义接口：interface interface-name{}
    2. 实现接口：class class-name implements interface-name{}
    3. 调用接口(接口引用指向实现接口的类的对象)：interface-name interfaceobj = class-name classobj
    4. 接口多态: 一个接口可以被多个类实现或被多个接口继承
    5. 嵌套接口(class-name.interface-name): 可以在类中声明接口(可以声明为public、private和protected)
    6. 默认接口方法：default void meth1(){},默认接口需要写方法体，默认方法可以被重写
    7. 接口多级继承：
      1. 接口可以通过关键字extends继承另一个接口，语法和继承类相同。如果类实现的接口继承自另一个接口，那么类必须实现继承链中定义的所有方法.
      2. 在所有情况下，类实现（类定义的接口方法）的优先级高于接口默认实现。
  2. 异常：
    1. try/catch
    2. try/catch/finally

接口（难点）:接口没有实例变量，也没有实现的方法体。一旦定义接口，任何数量的类都可以实现接口，同样，一个类可以实现多个接口。不相关的类可以实现同一个接口。

1. 定义接口：

  1. （使用关键字interface）

2. 实现接口:

  1. (使用关键字implements)

3. 调用实现接口方法：

  1. (通过接口引用指向实现类对象，再通过接口引用调用方法）
  2. （传入对应的参数匹配到相对应的方法）
  3. （接口引用只能调用实现的接口方法）

4. 接口的多态功能：

  1. （一个接口可以被多个类实现）
  2. （通过接口引用指向来调用指向类实现的接口方法）

5. 嵌套接口：

可以将接口声明为某一类或另一个接口的成员，这种接口被成为成员接口或嵌套接口。嵌套接口可以被声明为public、private和protected（顶级接口只有public和默认）。

6. 默认接口方法定义：（使用default关键字）：default return-type meth-name(parameter-list){}

7. 实现类可以重写default方法：

接口继承：（interface B implements A{}）

  1. 接口可以通过关键字extends继承另一个接口，语法和继承类相同。如果类实现的接口继承自另一个接口，那么类必须实现继承链中定义的所有方法
  2. 在所有情况下，类实现（类定义的接口方法）的优先级高于接口默认实现。

    1. 接口1->接口2(继承接口1，不重写接口1方法)->实现类：（类实现的优先级高于接口实现，通过接口引用调用方法，传入参数和接口方法一致，也只会调用类实现的方法）

2. 接口1->实现类<-接口2：(同样说明类实现高于接口实现)

3. 接口1—>接口2(重写接口1方法)->类(不重写接口方法)：（说明当类无实现时，子类实现优先级高于父类实现）（方法调用逐级向上查找）

异常处理

  1. 异常处理通过5个关键字进行管理：try、catch、throw、throws、finally。手动抛出异常：throw，从方法抛出任何异常：throws。一般形式：

1. 异常类型

所有异常类型都是内置类Throwable的子类。Throwable下有两个子类，1、Exception类。既可以用于程序应当捕获的异常，有可以用于创建自定义异常类型的子类。Exception有一个重要子类——RuntimeException。这种类型的异常是自动定义的。包括处理除零和无效数组索引这类情况。
2、Error类。不希望被程序捕获的异常，堆栈溢出就是Error类错误的一个例子。

2. 使用try和catch

  • 在try代码块中封装希望监视的代码，随后使用catch指定希望捕获的异常类型
  • 使用多条catch子句时，必须把异常子类写在异常超类之前。

    3. 嵌套的try语句

  • 如果内层try中没有catch能够捕获错误，传递到外层查询try语句的catch能否捕获

4. throw

  • throw一般形式如下：throw ThrowableInsatance；
    其中ThrowableInsatance必须是Throwable或其子类类型的对象。基本类型（int、char、String、Object）都不能用作异常。

ThrowError方法new构造了一个NullPointerException的异常实例，并将其返回给调用例程。main函数调用该方法，再次捕获

5. thrwos

  • 通过在方法声明中提供throws子句来列出方法可能抛出的异常类型（除了Error（不希望被捕获）和RuntimeException及其子类型异常）。

声明形式：meth-name() throws Error-name{ }

6. finally

  • 使用finally创建一个代码块，改代码块在执行完try/catch语句后执行，不管有无异常抛出，都会执行finally

在方法throw2中通过finally退出try代码块，在返回前执行finally子句。（换言之，finally代码块会在与之相关联的try代码块结束后执行。
