今日重点：java多态（重载和重写）

  1. 多态是指一个程序中同名的不同方法共存的情况。针对不同的情况来定义不同内容的方法。
  2. 重写：重写Overriding是父类与子类之间多态性的一种表现（一个接口，多种方法）
    假设在子类中定义某方法与其父类有同样的名称和參数，我们说该方法被重写 (Overriding)。当子类对象调用同名方法时，会调用到子类的方法而不会调用父类的方法，因为父类方法被隐藏了
  3. 重载：重载Overloading是一个类中多态性的一种表现。
    假设在一个类中定义了多个同名的方法，它们或有不同的參数个数或有不同的參数类型，则称为方法的重载(Overloading)。构造函数定义不同的参数个数和不同的参数类型也是重载

重载

1. 重载方法：一个方法，定义不同类型的参数，可以被多次匹配调用

方法形参与调用中的实参类型相匹配：

test()方法重载了4次，第一次为无参数，第二次为一个整型参数，第三次为两个整型参数，第四次为一个double参数。当调用重载的方法时，java会对调用方法的对象传入的实参和重载方法形参之间进行类型匹配。
但是这一匹配不总是准确的

因为Overload没有定义test(int)，所以当ob.test(i)调用重载时，就算i是int类型，但是因为没有与之匹配的重载方法，所以java自动把i转换成double类型来进行匹配。所以最终得到的结果是调用了test(double)方法

当执行new运算符时，会根据指定的参数的类型和数量来匹配到符合条件的重载构造函数，然后调用。多个重载能够应对多种实例化的需求

3. 参数传递：

  1. 值调用：（传递基本类型，如int、double等）

当对象调用方法时，传入的其实是实参的副本(复制一份然后把复制的那份传给方法形参)，方法再对这一个副本进行操作，所以值调用的方式无法影响实参。

  1. 引用调用：传递对象（传递引用数据类型，如：String就是一种引用数据类型和创建类时定义的引用数据类型）

当对象调用方法时，把对象本身作为参数传递给了方法，因为将对象本身作为参数传入了方法，所以在方法内修改对象将会影响作为实参的对象。

4. 返回对象

方法可以返回任何类型的数据(比如void返回空，自定义int、double等类型)，包括自己创建的类类型，如：Box volume(){},这种命名是合法的，表明返回的是Box类类型。

重写重载总结：重载是在一个类里根据不同情况构造该类的不同构造函数（参数类型不同，参数数量不同）来应对多种情况；在实例化类的对象时就起作用。而重写是基于在类的继承关系上，针对每一个类面对的情况而定义了同名但是参数类型不同（针对一类事件的不同情况）的方法，通过声明超类的引用变量来引用子类对象（超类引用变量可以引用子类对象），从而指定调用指定类中的同名方法。

    不能声明抽象的构造函数(不能abstract 类名{})，也不能声明抽象的静态方法(不能abstract static mehtod(){})，抽象类不能实例化对象
