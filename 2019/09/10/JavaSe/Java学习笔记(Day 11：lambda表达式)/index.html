lambda表达式（两个结构：lambda表达式和函数式接口)

1. lambda表达式

  1. lambda表达式(闭包)：本质上是一个匿名方法
  2. 实例：() -> 123.45：操作符 -> 是lambda操作符或箭头操作符。它将lambda表达式分成两个部分，左侧指定了lambda表达式所需要的所有参数(如果不需要则为空:()),右侧指定了lambda体，即lambda表达式要执行的动作。
  3. lambda体：一种包含单独一个表达式，另一种包含一个代码块。

2. 块lambda表达式

  1. 在块lambda中，可以声明变量、使用循环、指定if和switch、创建嵌套代码块。
  2. 在块lambda中必须显式使用return语句来返回值

3. 泛型函数式接口

4. lambda表达式作为参数传递：可以将lambda表达式作为参数传递，接受lambda表达式的参数的类型必须是与该lambda表达式兼容的函数式接口类型。

分析：

  1. stri()静态方法：第一个参数的类型为A，而A是一个函数式接口，所以这个参数可以接受任何A的实例引用（如：(str) -> str.toUpperCase()),第二个参数为String类型
  2. 调用stri()静态方法：outstr1 = stri((str) -> {},outstr);将lambda代码块作为第一个参数传入方法的形参（其类型由return result确定，返回的是一个字符串）。当lambda块很长，不适合作为参数嵌入到方法调用中时，可以把lambda赋给一个函数式接口变量，然后将这个接口变量作为参数传入方法

lambda方法引用(函数式接口):方法引用不提供方法的实现，他们引用一个已经存在的类或者对象的方法。

1. 静态方法引用: 方法(类目::静态方法名)

静态方法调用过程:
先调用stri()方法,MyStringops::strR(静态方法可以通过类名::方法名调用)作为A接口的实现对象传给sf,返回调用sf.func(s)

2. 实例方法引用：方法(类实例::实例方法)

方法引用的调用过程:
先调用getMethod()方法,ob::ObjR(实例方法需要通过类实例调用)作为实参传给sf,getMethod()方法返回调用sf.func()

泛型中的方法引用

在泛型类和泛型方法中也可以使用方法引用。

分析：

  1. stri(A f,T[] vals,T v)方法第一个参数是一个A函数式接口的对象，意味着A有且只有一个抽象方法func，而func的参数为(T类型参数的数组和T类型参数的变量v)，而且返回值为int，与MyArray类中的sfunc方法一致，只有它们名称不同，所以可以创建一个方法引用(MyArray::sfunc,指定T为Integer)，并将它作为stri方法的第一个参数
