SQL语句

基础语法规则

  1. 单行或多行书写，分号;结束
  2. 可以使用空格和缩进
  3. 不区分大小写，关键字建议使用大写
  4. –或#：单行注释语句，/。。。。/多行注释

SQL语句

1. DDL（CREATE、ALTER、DROP)(对数据库、表进行操作）

  1. 操作数据库（CRUD）
    1. C(Create):创建
      1. CREATE关键字
        1. CREATE database if not exists db1;//如果数据库不存在则创建，如存在则不创建(存在不报错)
        2. CREATE database db2 character set gbk;//设置数据库格式为gbk
        3. CREATE database if not exists db3 character set gbk;//判断db3是否存在并设置字符集为gbk
    2. R(Retrieve):查询
      1. SHOW关键字
        1. SHOW databases;//查看数据库
        2. SHOW create database mysql;//查看创建“mysql”数据库的语句
    3. U(Update):修改
      1. ALTER关键字
        1. ALTER database db3 character set utf8;//将db3的字符集类型改为utf8;
    4. D(Delete):删除
      1. DROP关键字
        1. DROP database if exists db3; //如果存在，删除指定数据库
    5. 使用数据库
      1. SELECT关键字
        1. SELECT database(); //查询当前使用的数据库名称
      2. USE关键字
        1. USE db1; //使用db1数据库
  2. 操作表
    1. C(CREATE)：创建
      1.形式：
      CREATE table table-name(
      列1 数据类型1,
列2 数据类型2,
....
列n 数据类型n           //最后一列不需要逗号
      );
      • SQL数据类型:
        1. int
        2. double:score double(5,2);//一共5位,小数点保留2位
        3. date:日期,形式:yyyy-MM-dd
        4. datetime:日期,形式:yyyy-MM-dd HH:mm:SS:DD
        5. timestamp:时间戳,形式:yyyy-MM-dd HH:mm:ss.(如不指定则使用当前时间)
        6. varchar:字符串,形式:name varchar(20);//最大20个字符
      1. CREATE table info( //创建table名为info的表
        ID int,
        name varchar(32),
        age int,
        score Double(4,1),
        birthday date,
        insert_time timestamp
        );
    2. R(Retrieve):查询
      1. SHOW关键字：
        1. SHOW tables;//查看所有表
      2. DESC关键字:
        1. DESC table-name; //查询表结构
    3. U(Update):修改
      1. 修改表名:
        1. ALTER table old-name rename to new-name;
      2. 修改字符集:
        1. SHOW create table Students;//先查看表属性
        2. ALTER table table-name character set utf8;//将字符集改为utf8;
      3. 添加一列:
        1. ALTER table table-name add 列名 数据类型;
      4. 改列名和数据类型:
        1. ALTER table table-name change 原列名 新列名 新数据类型; //change改列名和数据类型
        2. ALTER table table-name modify 列名 新数据类型; //modify只改数据类型
      5. 删除列:
        1. ALTER table table-name drop 列名;
    4. D(Delete):删除
      1. DROP table table-name;
      2. DROP table if exists table-name;

2. DML(INSERT、UPDATE、DELETE)(增删改语句)

  1. 添加数据
    1. insert into 表名(列名1,列名2,…列名n) values(值1,值2,….值n);
    2. 实例:insert int students(id,’name’,age,score,birthdat,insert_time) values(1,”ks-39”,20,90,”1222-12-01”,current_time);
    3. insert INTO stu(ID,phone_num) VALUES(1,”123456”),(2,”1234566”); //添加多个数据
  2. 删除数据
    1. delete from 表名 [where 条件] //如果不加条件则删除表中所有条件
    2. 实例：
      1. delete from students where id = 1; //删除id=1的列数据
      2. delete from students; //删除表格
    3. truncate table 表名; //删除表并创建一个和原来一样的空表
    4. 实例：
      1. truncate table students;//删除表格并创建一个一样的表格
  3. 修改数据
    1. update 表名 set 列名1 = 值1, 列名2 = 值2,…. [where 条件]
    2. 实例：
      1. update students set age = 20 where ID = 2; //将ID=2的列的age数值改为19
      2. update students set age = 20; //如果不加where则修改表中所有的age.

3. DQL(SELECT、WHERE、ORDER BY、GROUP BY)(查语句)

  1. 语法

    1. select 字段列表

    2. from 表名列表

    3. where 条件列表

    4. group by 字组字段

    5. having 分组之后的条件

    6. order by 排序

    7. limit 分页限定

4. DCL

  1. 管理用户
    1. 查询用户
      1. use mysql;
      2. select * from user;
    2. 创建用户
      1. create user ‘用户名‘@’主机名’ identified by ‘密码’;
    3. 删除用户
      1. drop user ‘用户名‘@’主机名’;
    4. 修改用户密码
      1. update user set password = password(‘新密码’) where user = ‘用户名’;
      2. set password for ‘用户名‘@’主机名’ = password(‘新密码’);
      3. 忘记了密码：
        1. （管理员）cmd——> net stop mysql
        2. 使用无验证方式启动mysql服务：mysqld –skip-grant-tables
        3. 新打开一个cmd：直接输入mysql回车登陆mysql
        4. 设置新密码：update user set password = password(‘新密码’) where user = ‘用户名’;
        5. 打开任务管理器，结束mysql进程
        6. （管理员）cmd——> new start mysql
        7. 使用新的用户名和密码登陆：mysql -u用户名 -p密码
  2. 权限管理
    1. 查询权限：
      1. show grants for ‘用户名‘@’主机名’;
        //usage表示可以登陆
    2. 授权s
      1. grant 权限类型 on 数据库.表名 to ‘用户名‘@’主机名’;
      2. GRANT SELECT ON db1.account to ‘test’@’localhost’; //test用户可以查看db1数据库的account表
      3. grant select,delete,update on db1.account to ‘test’@’localhost’; //将查看，删除，修改的权限赋给test用户
      4. grant all on . to ‘test’@’localhost’; //全权赋予test用户
    3. 撤销权限
      1. revoke 权限类型 on 数据库.表名 from ‘用户名‘@’主机名’;
      2. revoke select on * from ‘test’@’localhost’; //撤销test用户的查看权限
      3. revoke all on . from ‘test’@’localhost’; //撤销全部权限

约束：（保证数据正确性、有效性和完整性）

  1. 主键约束：（primary key）（非空且唯一）（一个表格只能有一个主键）
    1. 创建表时添加主键约束
      create table stu(
      id int primary key ,    //记得加空格,id数据必须非空且唯一
name varchar(32),
      );
    2. 删除主键约束
      alter table stu drop primary key; //不需要指定字段，因为表格只能有一个主键
    3. 添加主键约束
      alter table stu modify id int primary key; //如果表格数据有空数据和重复数据，需先更改或删除数据才能执行该语句
    4. 自动增长：如果字段数据类型为数值类型(int)，使用auto_increment自动增长
      1. 创建表是添加自动增长
        create table stu(
        id int primary key auto_increment;  //设置主键为int类型并自动增长，
        );
      2. 删除自动增长
        alter table stu modify id int; //保留主键约束，删除自动增长
      3. 添加自动增长
        alter table stu modify id int auto_increment;
  2. 非空约束：（not null）
    1. 创建表时对字段数据进行约束
      create table stu(
      id int,
name varchar(32) not null       //限制name不能为空
      );
    2. 删除非空约束：
      alter table stu modify name varchar(32); //modify更改name的数据类型
    3. 直接添加非空约束：
      alter table stu modify name varchar(32) not null; //modify更改name的数据类型加入not null限制
  3. 唯一约束：（unique）（null不做限制）
    1. 创建表时添加唯一约束
      create table stu(
      id int,
phone_number varchar(32) unique     //限制phonenumber为唯一
      );
    2. 删除唯一约束
      alter table stu drop index phone_number; //drop index 删除唯一约束
    3. 添加唯一约束
      alter table stu modify phone_number varchar(32) unique; //先将相同数据删除或修改才能执行
  4. 外键约束：（foreign key）（多表）（外键可以为null）
    1. 创建表时添加外键
      create table stu(
      ....
constraint 外键名 foreign key 外键列 reference 主表名(主表主键名)
      );
    2. 实例：
      1. 创建部门表
        create table department(
        id int primary key auto_increment,
dep_name varchar(20),
dep_location varchar(20)
        );
      2. 创建员工表
        create table employee(
        id int primary key auto_increment,
name varchar(20),
age int,
dep_id int,
constraint emp_dep foreign key (dep_id) references department(id)       //emp_dep为外键名,不可与字段重名.dep_id为外键,关联到department的id字段.
        );
      3. 添加部门信息
        insert into department values(null,”研发部”,”广州”),(null,”销售部”,”深圳”);
      4. 添加员工信息
        insert int employee(name,age,dep_id) values(“张三”,20,1),(“李四”,21,2),(“王五”,20,1); //1代表研发部，2代表销售部
      5. 无法进行以下操作：
        1. 删除department的研发部数据:delete from department where id = 1; //无法执行,因为employee表中的dep_id与之关联
        2. 添加不存在department表的id：insert into employee(name,age,dep_id) values(“www”,20,3); //无法执行，因为没有dep_id = 3的部门.
    3. 删除外键约束
      alter table employee drop foreign key emp_dep; //删除外键名emp_dep
    4. 添加外键约束
      alter table employee add constraint emp_dep foreign key (dep_id) reference department(id); //设置外键名为emp_dep，外键为dep_id，关联department表的id值
  5. 级联操作：外键随关联的主键而变动
    1. 添加外键时设置级联更新
      1. 先删除外键：
        alter table employee foreign key emp_dep;
      2. 再添加：
        alter table employee add constraint emp_dep foreign key (dep_id) references department(id) ON UPDATE CASCADE;
      3. 将department中研发部的id改为3，employee中与之关联的dep_id自动变为3
        1. update department set id = 1 where id = 3;
    2. 删除级联：
      1. 先删除外键：
        alter table employee foreign key emp_dep;
      2. 添加外键删除级联
        alter table employee add constraint emp_dep foreign key(dep_id) references department(id) on update cascade on delete cascade; //on delete cascade删除级联

多表查询

  1. 笛卡尔积：（取集合的所有组成情况）
    1. select * from department,employee; //两个集合的所有组成情况
  2. 内连接查询
    1. 隐式内连接：使用where提取数据
      1. select * from department,employee where employee.dep_id = department.id; //查询employee和department的对应情况
      2. select employee.name * from department,employee where employee.dep_id = department.id; //查询employee表的员工名称和department表的部门名
      3. select 
        t1.name,
t2.name
        from 
        employee t1,    //命名employee表为t1
department t2   //命名department表为t2
        where
        t1.dep_id = t2.id;
    2. 显示内连接：t1 [inner] join t2 on statement; //inner可选
      1. select * from employee inner join department on employee.dep_id = department.id; //将employee的数据与department的数据进行比对,提取出与dep_id对应的department数据
  3. 外连接查询
    1. 左外连接
      select 字段 from 表1 left [outer] join 表2 on 条件
    2. 右外连接
      select 字段 from 表1 right [outer] join 表2 on 条件
  4. 子查询
    1. 结果单行单列
      1. select * from employee where employee.salary = (select max(salary) from employee); //查询employee表中最高salary
      2. select * from employee where employee.salary < (select avg(salary) from employee); //查询工资低于平均工资的信息
    2. 结果多行单列
      1. select * from employee where dep_id in (select id from department where name = “研发部” or name = “销售部”); //查询employee表中dep_id对应department表中”研发部”和”销售部”的id。如1对应研发部，2对应销售部
    3. 多行多列
      1. select * from department t1,(select * from employee where employee.age > 20) t2 where t1.id = t2.dep_id; //查询年龄大于20的员工的信息和其对应部门信息

事务

  1. 操作：
    1. 开启事务：start transaction;
    2. 回滚：rollback;
    3. 提交：commit;
  2. 案例：
    start transaction; //开始一个事务,保证多行语句不会出现上一语句执行而下一语句因错误而无法执行的情况
    update account set balance = balance - 500 where name = “张三”;
    update account set balance = balance + 500 where name = “李四”;
    commit; //如果没有错误，提交事务的语句(即可以执行语句)
    rollback; //如果有错误，不执行事务中的语句
  3. 事务的四大特征
    1. 原子性：不可分割的最小操作单位，要么同时成功，要么同时失败(事务包含的语句要不就执行全部，要不就全都不执行)
    2. 持久性：当事务提交或回滚后，数据库会持久化地保存数据
    3. 隔离性：事务之间相互独立
    4. 一致性：事务操作前后数据总量不变。事务只做数据移动操作。
  4. 事务的隔离级别：
    1. 查询数据库隔离级别
      select @@tx_isolation;
    2. 设置隔离级别
      set global transaction isolation level 级别字符串
