Cookie:客户端会话技术,将数据存储到客户端

  1. 快速入门案例:

    1. 步骤:
      1. 创建Cookie对象,绑定数据:new Cookie(String name,String value);
      2. 发送Cookie对象:response.addCookie(Cookie cookie);
      3. 获取Cookies,拿到对象:Cookie[] cookies = request.getCookies();
      4. 遍历Cookies

  2. 原理分析:

    1. 客户端第一次请求:
      1. 客户端浏览器发送第一次请求给服务器,请求Test1的Cookie
      2. 服务器响应客户端的第一次请求,发送了响应头给客户端(内容:set-cookie:msg=hello)
      3. 客户端将msg=hello保存到客户端浏览器
    2. 客户端第二次请求:
      1. 客户端将请求头发送给服务器(cookie:msg=hello)
      2. 服务器接受客户端的请求头。(即接收msg=hello)

  3. 可以一次发送多个Cookie

  4. 设置Cookie的存活时间

    1. cookie.setMaxAge(30); //存活30s
    2. cookie.setMaxAge(-1); //服务器关闭后就删除
    3. cookie.setMaxAge(0); //立即删除

  5. 特点:Cookie数据存储在客户端浏览器

  6. 作用:在不登录的情况下完成服务器对客户端的身份验证

  7. 案例:记住上一次访问时间:

    1. 需求:
      1. 访问一个Servlet,如果是第一次访问,提示:您好,欢迎光临.如果不是第一次访问,提示:欢迎回来,你上次的访问时间为:
    2. 思路:
      1. Servlet服务器判断是否有lasttime的Cookie

Session:服务器端会话技术,在一次会话的多次请求间共享数据,将数据保存在服务器端的HttpSession对象中.

  1. 共享数据
    1. 获取Session对象:HttpSession session = request.getSession();
    2. 存储数据:session.setAttribute()
    3. 获取数据:Object getAttribute(String name);
  2. 原理:session的实现依赖与cookie(通过请求头和响应头实现)
  3. session的钝化和活化
    1. 钝化:在服务器正常关闭后，将session对象系列化到硬盘上
    2. 活化:在服务器启动后，将session文件转化为内存中的session对象

JSP:Java服务器端页面

  1. 脚本:
    1. <% code %>:在service方法内
    2. <%” code %>:直接输出在页面
  2. 内置对象:九大内置对象
    1. request:代表客户端的请求信息，主要用于获取客户端的参数和流。
      1. 方法
        (1) String getMethod() //获得提交方式
        (2) String getRequestURI() //获得请求的URL地址
        (3) String getProtocol() // 得到协议名称
        (4) String getServletPath() //获得客户端请求服务器文件的路径
        (5) String getQueryString() //获得URL的查询部分，post方法获得不到信息
        (6) String getServerName() //得到服务器的名称
        (7) String getServerPort() //获得服务器口号
        (8) String getRemoteAddr() //得到客户端的IP地址
        (9) String fetParameter(String name) //获得客户端传给服务器的name参数的值*
    2. response:对象和request是一对相应的内置对象，代表对客户端的响应
      1. 方法:
        (1) response.sendRedirect(目标页面路径); //重定向
        (2) response.setHeader(String,String); //设置HTTP头
    3. session:对象是由服务器自动创建的与请求相关的对象，服务器为每个用户都生成一个session对象，用于保存该用户的信息，跟踪用户的操作状态。session内部使用Map来保存数据
      1. 方法
        (1) session.setAttribute(String,Object); //给Object命名String，加入session
        (2) session.getAttribute(String); //取名为String的session的值
        (3) session.removeAttribute(String); //将名为String的内容从session中移除　
    4. application:将信息保存在服务器中，直到服务器关闭，否则application对象中保存的信息会整个应用中都有。
    5. out:输出到页面上(字符输出流)：out对象用于Web浏览器内输出信息，负责管理对客户端的输出。并且管理应用服务器上的输出缓冲区。在使用out对象输出数据时，可以对数据缓冲区进行操作，及时清理缓冲区中的残留数据。
    6. pagecontext:pageContext 对象的作用是取得任何范围的参数，通过它可以获取 JSP页面的out、request、reponse、session、application 等对象。pageContext对象的创建和初始化都是由容器来完成的，在JSP页面中可以直接使用 pageContext对象。
    7. config:主要作用是取得服务器的配置信息。
    8. page:page 对象代表JSP本身，只有在JSP页面内才是合法的。 它是java.lang.Object类的实例化对象。page隐含对象本质上包含当前 Servlet接口引用的变量，类似于Java编程中的 this 指针。
    9. exception:exception 对象的作用是显示异常信息，只有在包含 isErrorPage=”true” 的页面中才可以被使用，在一般的JSP页面中使用该对象将无法编译JSP文件。excepation对象和Java的所有对象一样，都具有系统提供的继承结构。
  3. 指令:用于配置JSP页面,导入资源文件
    1. 格式:<%@ 指令名称 属性名1=属性值1 属性名2=属性值2…%> 例如:<%@ page contentType=”text/html;charset=UTF-8” language=”java” %>
    2. 分类:
      1. page
        1. contentType:设置响应头
        2. import:导包
        3. errorPage:定义异常页面
      2. include:页面包含的导入页面的资源文件
      3. tglib:导入资源

JSP EL表达式

  1. 功能:处理MVC开发模式下JSP中的java代码

  2. 语法:

    1. ${3>4} //false

  3. 使用:

    1. 运算:

      1. 算数运算符:/(div) %(mod)
      2. 比较运算符:
      3. 逻辑运算符:&&(and) ||(or) !(not)
      4. 空运算符(判断对象是否为null):empty

    2. 获取值:

      1. 只能从域对象中获取值
      2. ${域名.键名}:从指定域中获取指定键值
        1. 域名:
          1. requestScope –> request
          2. sessionScope –> session

    3. 获取对象、List集合、Map集合的值

      1. 获取对象的值:${域名.键名.属性名}

      2. 获取List集合的值:${域名.键名[索引]}

      3. 获取Map集合的值:${域名.键名,key}
