MVC开发模式:模型(model)－视图(view)－控制器(controller)

  1. 原理:
    1. 控制器controller获取客户端浏览器的请求
    2. 控制器controller调用模型model进行业务操作
    3. 模型model将数据返回给控制器controller
    4. 控制器controller将数据给视图view
    5. 视图view将模型model的数据展示
  2. Controller(中转站.Servlet)
    1. 获取用户输入
    2. 调用模型
    3. 将数据转交给视图
  3. Model(业务逻辑操作.JavaBean)
    1. 完成业务逻辑.如:查询数据库，封装对象
  4. View(展示效果.JSP)
    1. 展示数据

三层架构:软件设计架构

  1. 流程:
    1. Servlet(Controller控制器)接收用户请求,接收参数,封装信息,调用业务逻辑层
    2. Service(Model模型)通过代码实现业务,调用数据访问层.将得到的数据返回给Servlet(Controller控制器)
    3. 当业务逻辑层访问时,将请求的数据返回给业务逻辑层
    4. 将数据存储,转发到JSP页面(View视图)
  2. 界面层(web层,SpringMVC框架):展示数据
    1. 功能:接收用户的参数,封装数据,调用业务逻辑层完成处理,转发到jsp页面完成显示
  3. 业务逻辑层(service层,Spring框架):处理业务逻辑。
    1. 功能:将dao层的简单CRUD操作组合成复杂的功能
  4. 数据访问层(dao层,MyBatis框架):操作数据存储文件。
    1. 功能:定义了简单的CRUD操作

业务流程:

  1. 需求:用户信息的增删改查(CRUD)

  2. 思路:

    1. 技术选型:Servlet(业务逻辑)+JSP(界面)+MySQL(数据访问)+JDBCTemplate(数据库封装)+Durid(数据库连接池)+BeanUtils(封装数据工具类)+Tomcat(web服务器)

    2. 数据库设计:创建目标数据库和目标表

    3. 开发:

      1. 环境搭建:
        1. 创建数据库环境
        2. 创建项目,导入jar
      2. 编码

    4. 测试

    5. 部署运维

资源:

  1. domain包:
    1. User类:处理数据库数据,getter和setter方法,toString()方法
    2. PageBean类:处理分页查询显示功能,包含totalCount信息总数、totalPage总页数、List list每页显示数据、currentPage当前页数、rows每页显示信息数等变量的getter和setter方法
  2. Utils工具包:Druid数据库连接池
  3. View视图:web层(SpringMVC框架,Servlets):与前端jsp通过request和response进行数据交互
    • 简单功能:登录、增删改查
    1. LoginServlet:(校验验证码和用户名密码)(设置编码utf-8)
      1. 校验验证码:
        1. request.getSession()获取Session
        2. Session.getAttribute(“checkcode”)获取服务器CheckCodeServlet生成的验证码,将其转换为String
        3. 忽略大小写比较,错误则转发(forward,地址栏不变)回当前页面,正确则校验用户名和密码
      2. 校验用户名和密码
        1. 通过request.getParamemeterMap()获取login.jsp页面的文本框的输入为映射形式(jsp页面的文本框的name必须与User类中定义的相对应,不然无法获取),使用BeanUtils.populate(user,map)封装为映射,保存到Bean.再通过调用service层的login(user),借由DaoImp类的查询方法查询数据库信息.
        2. 判断是否登陆成功,成功则保存用户名和密码到session,重定向(dispatcher,跳转到另一页面,地址栏发生变化)到UserListServlet页面,否则设置提示信息:request.setAttribute(“msg”,”error”),跳转(forward)回当前页面
    2. UserListServlet:(显示用户信息)
      1. 调用service的findAll()方法,findAll()方法再调用dao层的findAll()方法查询数据库表中的所有数据,返回一个list集合:List users = service.findAll()
      2. 将users存入request:request.setAttribute(“users”,users);
      3. 转发forward到list.jsp:request.getRequestDispatcher(“/list.jsp”);
    3. AddUserServlet:(添加用户信息功能)(设置编码utf-8)
      1. 获取add.jsp添加的参数,存入映射中:request.getParameterMap()
      2. 使用BeanUtils.populate(user,map)封装映射到Bean.
      3. 在调用service层的addUser(user)方法,addUser()方法再调用dao层的addUser(user)方法将数据存入数据库
      4. 重定向回UserListServlet页面:response.sendRedirect(request.getContextPath()+”/UserListServlet”).forward(request,response);
    4. DeleteUserServlet:(删除用户信息功能)
      1. 获取list.jsp页面delete的数据的id值:request.getParameter(“id”);
      2. 调用service的deleteUser(id)方法,deleteUser(id)方法再调用dao层的deleteUser(parseInt)方法将对应id的信息删除
      3. 重定向回UserListServlet页面:response.sendRedirect(request.getContextPath()+”/UserListServlet”).forward(request,response);
    5. UpdateUserServlet:(更改用户信息功能)(设置编码为utf-8)
      1. 将需要更改的信息回显在update.jsp
        1. EchoUserServlet:(将需要修改的信息回显在页面上)
          1. 获取修改的id值:request.getParameter(“id”)
          2. 调用service的echoUser(id)方法,echoUser(id)方法再调用dao层的echoUser(parseInt)方法,返回相应id的数据
        2. 获取update.jsp页面修改的信息,存入Map映射:request.getParameterMap();
        3. 将映射存入Bean:BeanUtils.populate(user,map)
        4. 调用service层的updateUser(user)方法:service.updateUser(user),updateUser(user)再调用dao层的updateUser(user)方法,修改相应id的数据
        5. 重定向回UserListServlet:response.sendRedirect(request.getContextPath()+”/UserListServlet”).forward(request,response);
    • 复杂功能:
    1. DeleteSelectUserServlet:(删除选中用户)
      1. 获取选取的所有id:String[] ids = request.getParameterValues(“selectId”);
      2. 调用service的deleteSelect(ids)方法,deleteSelect(ids)遍历ids数组,每一id再调用dao层的deleteUser(Integer.parseInt(id))方法,主意删除
      3. 重定向回UserListServlet:response.sendRedirect(request.getContextPath()+”/UserListServlet”).forward(request,response);
    2. FindUserByPageServlet:(分页显示复杂条件查询)(设置编码)
      1. 获取currentPage(当前页面)和rows(每页显示数):request.getParameter(“currentPage”)/request.getParameter(“rows”);
      2. 剔除currentPage和rows,初次访问时给currentPage和rows赋初值
      3. 获取条件查询的参数:request.getParameterMap();
      4. PageBean对象调用service的findUserByPage(currentPage,rows,condition),findUserByPage(currentpage,rows,conditon)调用dao层的findTotalCount(condtion)方法,获取总记录数:pb.setTotalCount(dao.findTotalCount(condtion))
      5. findUserByPage()方法调用dao层的findByPage(start,rows,condition)方法,获取查询的数据
      6. 将PageBean存入request:request.setAttribute(“pb”,pb);
      7. 将condition存入request:request.setAttribute(“condition”,condition);
      8. 转发回当前页面:request.getRequestDispatcher(“.list.jsp”);
  4. Controller控制器:(service层,Spring框架)(面向接口编程):接受View视图的调用,调用dao层的对应方法操作数据库
    1. UserService接口
    2. UserServiceImpl实现类:

      2. public User login(User user){};//查询数据库返回查询结果
      3. public void addUser(User user){};//将数据添加到数据库
      4. public void deleteUser(String id){};//将对应id的数据删除
      5. public Usser echoUser(String id){};//返回对应id的数据
      6. public void updateUser(User user){};//将数据更新到数据库
      7. public void delteSelectUser(String[] ids){};//将获取的ids数据遍历删除
      8. public PageBean findUserByPage(String _currentPage,String _rows,Map<String, String[]> condition){};//查询数据库总记录数和实现分页查询.
  5. Dao层,MyBatis框架:(接收service的调用并执行数据库操作)(面向接口编程)
    1. 声明JdbcTemplate对象
    2. public List findAll(){}//执行查询数据库操作:select * from user;返回List集合:template.query()
    3. public User UserLogin(String u,String p){}//执行查询username和password操作:select * from user where username=? and password=?;返回一个User对象:template.queryForObject()
    4. public User addUser(User user){}//执行添加数据:insert into user values(null,null,null,?,?,?,?,?,?);

    7. public void updateUser(User user){}//执行修改指定id的数据:update user set name = ?,gender = ?,age = ?,address = ?,qq = ?,email = ? where id = ?;//
    8. public int findTotalCount(Map<String, String[]> condition){}//复杂查找的记录数:select count(*) from user where 1 = 1;返回int对象
