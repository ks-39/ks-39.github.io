内容:

  1. 网路通信基础概念
  2. 服务器概念
  3. 服务器软件(Tomcat为例):可以部署web项目,让用户通过浏览器访问这些项目
  4. Servlet:是服务器或应用服务器上的程序
  5. HTTP:定义了客户端和服务器的交互的数据格式
  6. HTTP对象
    1. Request请求对象:request对象是服务器对浏览器请求的封装,request用来取出请求信息.
    2. Response响应对象:response是服务器对服务器响应的封装,response则用来添加要返回给浏览器的信息.
    3. ServletContext对象:(和服务器进行通信)

网络通信三要素

  1. IP:电子设备(计算机)网络的唯一标识
  2. 端口:应用程序在计算机的唯一标识.(0-65535)(不取1024以内端口)
  3. 传输协议:规定数据传输的规则
    1. TCP:安全协议，三次握手，速度较慢
    2. UDP:不安全协议，速度快

服务器概念:

  1. 服务器:安装了服务器软件的计算机(高配电脑)
  2. 服务器软件:接收用户请求，处理请求。

web服务器软件:可以部署web项目,让用户通过浏览器访问这些项目(Tomcat为例)

Tomcat:中小型JavaEE服务器

  1. 启动:startup.bat/sh
    1. 可能出现的问题:
      1. 黑窗口一闪而过:JAVA_HOME配置
    2. 启动报错:
      1. 找到占用的端口，结束进程
        1. cmd:netstat -ano
        2. 找到8080端口,记下PID,
        3. 启动任务管理器,结束占用8080端口的PID进程
      2. 修改自身端口
        1. Tomcat的conf文件夹下server.xml文件
        2. 修改Connector port
  2. 访问:
    1. localhost:8080
    2. ip:8080
  3. 关闭
    1. shutdown.bat/sh
    2. 在startup.bat窗口下ctrl+c
  4. 配置
    1. 部署项目方式
      1. 直接将项目放到webapps目录下,在浏览器中输入:localhost:8080/目标文件夹/目标文件
      2. 将html文件打包成war文件(文件名不能有中文),放置到webapps文件夹下,自动解压
      3. 热部署:在conf/Catalina/localhost创建xml文档,编写
    2. 动态项目有目录结构
      1. 项目根目录
        1. WEB-INF目录
          1. web.xml:web项目核心配置文件
          2. classes目录:放置字节码文件
          3. lib:放置jar
  5. 将Tomcat集成到IDEA中,创建JavaEE项目
    1. 部署Tomcat:Run/Edit configurations/Tomcat server
    2. 自动部署新创建文件:Run/Edit configurations/on Update action:Update Resources

Servlet

  1. 概念:运行在服务器端的小程序。

  2. 本质:是一个接口,定义了Java类被tomcat识别的规则）

  3. 使用方式:实现servlet:自定义类，实现servlet接口，复写方法

  4. 简单部署:

    1. 创建JavaEE项目
    2. 定义实现类，实现Servlet接口:public class ServletDemo1 implements Servlet
    3. 实现接口的抽象方法
    4. 配置Servlet
      1. 配置web.xml文档

  5. Servlet执行原理

    1. 当服务器接收到客户端的请求后(即启动Tomcat),解析URL路径,获取访问的Servlet资源路径
    2. 查找web.xml文件,将url-pattern标签体内容的类的字节码文件(…Class.forName())加载进内存
    3. 创建对象:constructs.newInstance()
    4. 调用方法:service()

  6. Servlet方法的生命周期：

    1. 创建:public void init(ServletConfig servletConfig){} //初始化方法,在访问服务器时执行,只执行一次
      1. 默认情况第一次被访问时创建
      2. 更改创建时机,服务器启动时创建:web.xml中标签体中添加正整数0-10
      3. Servlet在内存中只存在一个对象（单例）
        1. 多线程访问的安全问题
        2. 不要在Servlet中定义成员变量（在方法体中定义局部变量 ）
    2. public void service(ServletRequest servletRequest, ServletResponse servletResponse){} //服务方法,每次访问服务器都会执行
    3. public void destroy(){} //销毁方法,在关闭浏览器时执行

1. Servlet配置文件(使用注解)

  1. 步骤

    1. 创建JavaEE项目于(Servlet版本3.0以上)
    2. 定义实现类，实现Servlet接口
    3. 复写方法
    4. 在类上使用@WebServlet注解，进行配置@WebSerlet(“资源路径”)

  2. 访问路径定义规则:

    1. /xxx:”/test1”
    2. /xxx/xxx:”:”/user/test5”,访问时必须写全
    3. .do:”/“,无论写什么路径都可以访问,访问级低

2. Servlet体系结构

  1. GenericServlet抽象类实现了Servlet接口

    1. HttpServlet抽象类

  2. GenericServlet:只需复写service方法,其他接口方法默认为抽象方法,如需要，手动添加其他方法

  3. HttpServlet:对http协议封装,复写doGet和doPost方法

HTTP基本概念:定义了客户端和服务器的交互的数据格式

  1. 基于TCP/IP的高级协议
  2. 默认端口80
  3. 基于请求/响应模型：一次请求一次响应
  4. 无状态:每次请求相互独立.不能交互数据

1. Request和Response对象:服务器创建

  1. 概念:request对象是服务器对浏览器请求的封装，而response是服务器对服务器响应的封装。request用来取出请求信息，而response则用来添加要返回给浏览器的信息。
  2. 生命周期:
    1. Tomcat服务器根据url的资源路径,创建对应的ServletTest类的对象
    2. Tomcat服务器创建Request和Response对象,Request对象封装请求数据
    3. Tomcat将Request和Response对象传递给Service方法,调用Service方法.
    4. 通过Request对象获取请求的数据，通过Response对象设置响应数据
    5. 服务器在给浏览器做出响应之前，会从Response对象中拿出设置好的响应数据

2. HTTP请求消息

  1. 请求行格式：请求方式+请求url+请求协议/版本
    1. GET user/hello.html HTTP/1.1
    2. 请求方式:GET和POST
      1. GET请求参数在请求行中，长度有限制
      2. POST请求参数在请求体中，长度无限制
  2. 请求头格式：请求头名称：请求头值
    1. User-Agent:浏览器和浏览器版本
    2. Accept:表明可以接收的信息格式
    3. Referer:防盗链和统计工作。表明当前页面的来源
  3. 请求空行：空行
  4. 请求体:

3. Request对象:客户端的请求,当客户端通过HTTP协议访问服务器时,HTTP请求头中的所有信息都封装在这个对象中,通过这个对象提供的方法，可以获得客户端请求的所有信息。

  1. 功能:

    1. 获取请求行:GET /user/test?…. HTTP/1.1

      1. String getMethod():获取请求方式:GET
      2. String getContextPath():获取虚拟目录:/user
      3. String getServletPath():获取Servlet路径:/test1
      4. String getRequestURI:获取请求URL:/user/test
      5. String getProtocol():获取协议及版本:HTTP/1.1
      6. String getRemoteAddr():获取客户机IP

    2. 获取请求头:user-agent:….

      1. String getHeader(String name):通过请求头名称获取请求头数据
      2. 防盗链：

    3. 获取请求体:只有POST请求方式才有请求体

      1. BufferedReader getReader():获取字符流输入
      2. ServletInputStream getInputStream():获取字节流输入
      3. 获取文本框输入:

    4. 获取请求参数通用方式:

      1. String getParameter(String name):根据参数名获取参数值
      2. 中文乱码
        1. 设置编码:request.setCharacterEncoding(“utf-8”);

  2. 案例1:请求转发：服务器内资源跳转(getRequestDispatcher(“目标”).forward())

  3. 案例2:登陆

    1. 思路:

      1. 编写login页面:username和password输入框
      2. 使用Druid数据库连接池技术，操作mysql数据库中的user表
      3. 使用JDBCTemplate封装JDBC
      4. 登陆成功跳转到展示页面
      5. 登陆失败跳转到失败页面

    2. 步骤:

      1. 创建项目,在web文件夹下创建login.html页面,导入配置文件，jar包

      2. 创建数据库login,创建user表

      3. 创建实体类User.定义表格属性的getter和setter方法

      4. 创建JDBCUtils工具类.使用Druid连接池

      5. 创建控制表格数据的类:UserDao.定义登陆方法,使用JdbcTemlpate的query方法执行select语句

      6. 控制台测试:创建test方法.声明User类对象(调用set方法)和UserDao类对象(调用login方法).使用@test注解执行test方法

      7. 创建LoginServlet类

4. HTTP响应消息

  1. 响应行:HTTP/2.0 200 OK;(HTTP协议)(2.0版本)(状态码200)(状态码含义)
    1. 状态码分类
      1. 1xx:服务器等待完全接收
      2. 2xx:响应成功
      3. 3xx:302重定向,304访问缓存
      4. 4xx:客户端错误。404无资源。405无do方法
      5. 5xx:服务器错误。
  2. 响应头
    1. Content-Type:text/html;charaset=UTF-8. //响应体数据格式以及编码格式
  3. 响应空行:空行
  4. 响应体:传输的数据

Response对象:Web服务器收到客户端的http请求,会针对每一次请求,分别创建一个用于代表请求的request对象、和代表响应的response对象。request和response对象即然代表请求和响应，那我们要获取客户机提交过来的数据，只需要找request对象就行了。要向客户机输出数据，只需要找response对象就行了。

  1. 功能

    1. 设置响应行:HTTP/1.1 200 ok
      1. 设置状态码:setStatus(int status)
    2. 设置响应头:setHeader(String name,String value)
    3. 设置响应体:
      1. 获取输出流:
        1. PrintWriter getWriter();//字符流
        2. ServletOutputStream getOutputStream();//字节流
      2. 将数据输出到客户端

  2. 案例1:重定向(设置状态码和设置头)

    1. 步骤
      1. 设置状态码:response.setStatus(302);
      2. 设置响应头:response.setHeader(“location”,”/…/…)
        或:response.sendRedirect(“/…/…”);
    2. 重定向特点（redirect）
      1. 地址栏变化
      2. 可以跳转到别的站点,访问别的站点资源
      3. 两次请求(重定向302一次,新页面状态码一次),不能使用request对象来共享数据
    3. 转发特点（forward）
      1. 地址栏不变
      2. 只能访问当前站点服务器的资源
      3. 转发是一次请求,可以使用request对象来共享数据
    4. 绝对路径:
      1. 客户端使用:需要加虚拟目录
      2. 服务器使用:不需要加虚拟目录
      3. 动态获取虚拟目录:getContextPath()

  3. 案例2:服务器输出字符数据/字节数据到浏览器(PrintWriter getWriter()/ServletOutputStream getOutputStream())

    • 字符流
    1. 设置响应头的解码格式,限制浏览器解码格式:
      response.setContentType(“text/html;charset=utf-8”)
    2. 获取字符输出流:PwintWriter pw = response.getWriter()
    3. 输出字符数据:pw.println()
    • 字节流
    1. 获取字节输出流:ServletOutputStream sos = response.getOutputStream();
    2. 输出字节数据:sos.write(“”).getBytes(“charset”);

1. ServletContext对象：(和服务器进行通信)

  1. 获取ServletContext对象:
    1. request.getServletContext()
    2. 继承HttpServlet。:this.getServletContext()
  2. 功能:
    1. 获取MIME类型(通信过程的文件数据类型)(响应头Content-Type:text/html):String getMimeType(String file);
    2. 共享数据(Test2设置(保存)数据,Test3获取数据)
      1. setAttribute(String name,Object value)
      2. getAttribute(String name)
      3. removeAttribute(String name)
    3. 获取文件真实路径(服务器路径):String getRealPath(String path)
      1. web目录:String getRealPath(“/..txt”)
      2. WEB-INF目录:String getRealPath(“/WEB-INF/.txt)
      3. src目录:String getRealPath(“/WEB-INF/classes/..txt);
  3. 案例:文件下载
    1. 需求:
      1. 页面有超链接
      2. 点击超链接弹出下载
      3. 完成图片下载
    2. 思路:
      1. 任何资源都必须要弹出下载提示框
      2. 使用响应头设置资源打开方式:content-dispostion:attachment;filename=xxx
    3. 步骤:
      1. 创建html页面,编辑href属性,指向目标Servlet,传递资源参数filename
      2. 创建Servlet
        1. 获取文件名称:String getParameter(“filename”);
        2. 获取ServletContext对象:ServletContext getServletContext();
        3. 获取文件真实路径:String getRealPath(“/../“);//web目录下的文件
        4. 使用字节输入流加载文件进内存:FileInputStream(path)
        5. 设置response响应头
          1. 获取文件类型:String getMimeType(filename)
          2. 设置Content-Type:resp.setContentType(filetype)
          3. 将中文文件名解码:URLEncoder.encode(filename,”utf-8”)
          4. 设置Content-disposition:resp.setHeader(“content-disposition”,”attachment;filename=”+fileNameEncoder”)
        6. 写出数据到response输出流:ServletOutputStream getOutputStream();
        7. 释放资源:sos.close()
