Spring简介:

  1. Spring是一个轻量级控制反转(IoC)和面向切面(AOP)的容器框架。
  2. Spring使用的是基本的JavaBean来完成以前只可能由EJB完成的事情。然而，Spring的用途不仅仅限于服务器端的开发。从简单性、可测试性和松耦合性角度而言，绝大部分Java应用都可以从Spring中受益。

    Spring优点:(创建和处理对象)(集成框架)(封装API)(集成Junit4)(AOP切面拦截)(配置文件管理业务逻辑)

  3. 方便解耦,简化开发:Spring 就是一个大工厂,可以将所有对象的创建和依赖关系的维护交给 Spring 管理。
  4. 方便集成各种优秀框架:Spring 不排斥各种优秀的开源框架,其内部提供了对各种优秀框架(如 Struts2、Hibernate、MyBatis 等)的直接支持。
  5. 降低JavaEE的API的使用难度:Spring对JavaEE开发中非常难用的一些API(JDBC、JavaMail、远程调用等)都提供了封装，使这些API应用的难度大大降低。
  6. 方便程序的测试:Spring支持JUnit4,可以通过注解方便地测试Spring程序。
  7. AOP编程的支持:Spring提供面向切面编程,可以方便地实现对程序进行权限拦截和运行监控等功能。
  8. 声明式事务的支持:只需要通过配置就可以完成对事务的管理，而无须手动编程。

      IoC(控制反转):

  15. Spring中将IoC容器管理的对象称为Bean，这个和JavaBean并没有什么关系，就跟Java和JavaScript一样。
  16. 为了实现IoC功能，Spring提供了两个类
    1. BeanFactory：Bean工厂，借助于配置文件能够实现对JavaBean的配置和管理，用于向使用者提供Bean的实例。
    2. ApplicationContext：ApplicationContext构建在BeanFactory基础之上，提供了更多的实用功能。

      DI(依赖注入):创建对象的时候，向类里面的属性设置值。我们使用Spring是用他管理对象之间的依赖关系（通过配置文件），这就是经常听到的依赖注入机制。这个注入的关系通过IoC容器进行管理。IoC容器中管理的就是一个个Bean包含的对象

      核心容器:Spring框架中提供了两个核心容器：BeanFactory容器和ApplicationContext容器。

  17. BeanFactory容器:这是一个最简单的容器，它主要的功能是为依赖注入提供支持, 在 Spring 中，有大量对 BeanFactory 接口的实现。其中，最常被使用的是 XmlBeanFactory 类。这个容器主要是通过xml文件来读取配置信息的，从而装配Bean。
  18. ApplicationContext容器(常用):ApplicationContext是BeanFactory的子接口,也称为应用上下文,不仅包含了BeanFactory的所有功能,还添加了对国际化,资源访问,事件传播等方面的支持.有两种创建ApplicatoinContext接口实例的方法.
    1. 通过ClassPathXmlApplicationContext创建 通过类路径classPath中寻找指定的XML配置文件,找到并装载完成ApplicationContext的实例化工作.
    2. 通过FileSystemXmlApplicatoinContext创建 通过指定的文件系统路径(绝对路径)中寻找指定的XML配置文件,找到并装载完成ApplicationContext的实例化工作.

      Bean的配置:

      通常使用xml配置文件。通常为一个Bean配置id或name和class两个属性即可,注意:如果Bean中未指定id和name,则Spring会将class值当做id使用.

Bean作用域

  1. Singleton(单例):每次getBean()都只能得到同一个对象
  2. prototype(原型):每次getBean()获得的对象不同

Bean装配(重点):(Bean依赖注入)(Bean类提供默认构造函数)(Bean类要有setter方法)

XML装配

Annotation装配(常用):通过注解(Annotation)来实现Bean的装配工作可以解决XML配置文件过于臃肿的问题,且便于后期维护.

  1. 常用注解:

    1. @Respository:将数据访问层(DAO层)的类标识为Spring中的Bean.

    2. @Service:将业务逻辑层Service类标识为SpringBean

    3. Controller:将控制层Controller类标识为SpringBean

    4. Autowired:用于Controller层自动装载Bean
