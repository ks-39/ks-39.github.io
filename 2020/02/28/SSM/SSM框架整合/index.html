技术点:

  1. 服务器: Tomcat 9.0.27
  2. 数据库: Mysql
  3. 前端框架:EasyUI
  4. 总结技术: JSP+jQuery+Ajax+EasyUI+Spring+Spring MVC+MyBatis+MySQL+Maven

1. 环境配置

1. 导入依赖

  1. mysql
  2. mybatis/mybatis-spring/pagehelper
  3. druid
  4. spring-core/spring-context/spring-web/spring-beans/spring=jdbc/spring-tx/spring-aop/spring-context-support/spring-aspects/spring-webmvc
  5. javax-servlet-api
  6. jstl
  7. jackson-annotations/jackson-core/jackson-databind
  8. jqeury

2. 建表

2. Java代码

1. Bean层:根据数据库字段创建Bean

2. Dao层:创建接口Mapper

3. Service层:创建mapper接口的同名方法

4. Service实现层(@Service/@Autowired):实现Service的接口方法

5. Controller层(@Controoler/@Autowired):控制前台的增删改查、页面跳转操作(@RequestMapping/@ResponseBody)

3. Resources资源

1. database.properties(数据库信息)

2. Mybatis_config.xml:(Mybatis配置文件,由于是ssm整合,所以仅需要基本模板)

3. Mapper映射文件(SQL语句)

4. Spring与Mybatis的整合：(applicationContext.xml)

  1. 开启Dao层扫描器
  2. 开启IOC注解扫描器
  3. Spring和Mybatis整合
    1. 配置连接池
    2. 配置SqlSessionFactory
  4. 开启事务注解扫描
    1. 配置Spring事务管理器
    2. 配置事务管理器

5. SpringMVC配置:SpringMVC.xml

  1. 开启SpringMVC注解
  2. 扫描Controller层注解
  3. 视图解析器(前缀和后缀)
  4. 过滤静态资源
  5. 拦截器

6. web.xml配置(前端页面控制器)

  1. 配置前端控制器DispathcerServlet
  2. 配置解码器
  3. 监听spring
  4. 默认页面设置

总结：

1. 前端方面注意点:

  1. function不能嵌套function

  2. ajax请求的success函数”msg”和”success”统一由后台发回

  3. 函数书写方式:function name(){…..}

  4. ajax的dataType: 需要后台返回json数据时指定为json

  5. ajax获取流程:

    1. 前台代码name属性 —> Bean实体类的变量 —> 数据库字段
    2. 因此尽量使前端name属性与Bean实体类的对应变量同名、

  6. 页面url路径变换:top.location.href=”全url路径”

2. 后端方面注意点:

  1. pagehelper: page和rows参数(必须是rows参数)

  2. ModelAndView:

    1. addObject(String,Object)
    2. setViewName(String)

  3. 消除全部session: session.invidate();

  4. requestDispatcher.forward()必须写全路径

  5. return: “redirect:Url路径 “执行转发

  6. tomcat的URL和web.xml的welcome-file冲突：将tomcat的Deployment的Application context改为“/”
